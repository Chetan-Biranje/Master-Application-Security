# 🛡️ 365 Days of Application Security

> A free, structured, day-by-day roadmap to go from zero to job-ready Application Security Engineer.
> No paid courses required. Everything here is free or low-cost.

## 👤 Who This Is For

- Complete beginners who want to enter Application Security
- Developers who want to understand how their code gets attacked
- Bug bounty hunters who want a structured foundation
- IT professionals switching to cybersecurity

**Prerequisites: None. Start from Day 1.**

## 🗺️ Roadmap at a Glance

| Phase | Days | Topic | Time/Day |
|---|---|---|---|
| [Phase 1](./phase-1-foundations.md) | 1–60 | Foundations — Networking, HTTP, Linux, OWASP | 1–2 hrs |
| [Phase 2](./phase-2-web-security.md) | 61–130 | Web Security — XSS, SQLi, CSRF, IDOR, Auth | 2 hrs |
| [Phase 3](./phase-3-api-security.md) | 131–190 | API Security — REST, JWT, OAuth, GraphQL | 2 hrs |
| [Phase 4](./phase-4-tools.md) | 191–240 | Tools — Burp Suite, Nmap, ffuf, SQLmap | 2 hrs |
| [Phase 5](./phase-5-devsecops.md) | 241–290 | DevSecOps — SAST, DAST, CI/CD, Secrets | 2 hrs |
| [Phase 6](./phase-6-bug-bounty.md) | 291–335 | Bug Bounty — Recon, Reporting, Platforms | 2–3 hrs |
| [Phase 7](./phase-7-advanced.md) | 336–365 | Advanced — Threat Modeling, Code Review, Career | 2–3 hrs |

## 📋 How to Use This Repo

1. Start at **Day 1** in Phase 1
2. Each day has a task, resources, and a lab
3. Check off tasks as you go (fork the repo and edit your own copy)
4. Don't skip — each phase builds on the previous one
5. Take notes — create a private repo for your writeups
